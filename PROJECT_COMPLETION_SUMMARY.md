## 🏗️ Architecture Patterns

### Established & Proven

**Svelte 4 Pattern:**
```svelte
<script lang="ts">
  export let size: 'sm' | 'md' | 'lg' = 'md';
  const sizeStyles = { sm: 'var(--renge-space-1)', md: 'var(--renge-space-2)' };
</script>
<button style="padding: {sizeStyles[size]}; {style}" on:click {...$$restProps}>
  <slot />
</button>
```

**Vue 3 Pattern:**
```vue
<script setup lang="ts">
const props = withDefaults(defineProps<Props>(), { size: 'md' });
const computedStyle = computed(() => ({ padding: sizeStyles[props.size] }));
</script>
<template>
  <button :style="computedStyle" v-bind="$attrs">
    <slot />
  </button>
</template>
```

**Key principles applied across all 124:**
- ✅ CSS custom properties only (--renge-space-*, --renge-color-*, etc.)
- ✅ Golden ratio (PHI) integration
- ✅ Fibonacci sequences for animations/spacing
- ✅ Framework-agnostic token consumption
- ✅ Full TypeScript strict mode
- ✅ Accessibility (ARIA labels, semantic HTML, keyboard nav)
- ✅ No hardcoded values
- ✅ Consistent prop APIs across frameworks
